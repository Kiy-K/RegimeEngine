# Contributing to RegimeEngine

We welcome contributions to RegimeEngine. To ensure a smooth process, please follow these guidelines.

## Code of Conduct

Maintain a professional and respectful environment when interacting with other contributors and maintainers.

## How to Contribute

1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Implement your changes.
4. Ensure your code follows PEP 8 guidelines.
5. Submit a pull request with a clear description of your changes.

## Development Style

- Use PEP 8 for Python code formatting.
- Include docstrings for new modules, classes, and functions.
- Write unit tests for new features.
- Ensure all tests pass before submitting a pull request.

## Reporting Issues

Use the GitHub issue tracker to report bugs or suggest enhancements. Provide a clear description and steps to reproduce any reported bugs.
